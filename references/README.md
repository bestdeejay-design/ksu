# Справочник дизайнера

> Отдельный проект, выделенный из портфолио Ксении (`ksu/`).
> Основа — референсы, которые уже были собраны в портфолио
> (см. `references.js` и `references.css` в этой папке — это
> вырезанный как есть код страницы).

## Что это такое

Чисто визуальный справочник + путеводитель «куда идти за референсами».
Никакой лишней теории — только картинки, ссылки и быстрый доступ.

## Когда нужен

- **Нет идей.** Застой, выгорел, пустая голова — открыл и сразу
  увидел десятки направлений и живых примеров.
- **Надо быстро скинуть заказчику референсы по нескольким проектам
  сразу.** Вместо того чтобы собирать ссылки по кусочкам в переписке,
  формируешь подборку по категориям и отправляешь одной ссылкой.

## Принципы

- Проект **чисто визуал**: минимум текста, максимум превью и ссылок.
- Это **справочник куда идти**: каждая карточка ведёт на реальный
  источник (Behance / Dribbble / Figma / Pinterest) или на сервис
  для мудбордов (Are.na / Muzli / Designspiration).
- База — референсы из портфолио `ksu`. Их можно дополнять и
  расширять под новые задачи.

## Текущее состояние (на момент выноса)

В папке лежат вырезанные куски из портфолио:

- `references.js` — данные (`refsData`, `specialResources`,
  `boardResources`) и функции рендера (`buildReferences`,
  `buildResourceSection`), а также i18n-ключи `refs.*`.
- `references.css` — стили блока References (`.references`, `.ref-card*`,
  `.ref-mini*`) и их адаптив.
- `README.md` — этот документ.

Код пока **не собран в рабочую страницу** — это просто вынесенный
кусками исходник. Чтобы сделать его живым, нужна отдельная
HTML-страница, которая подключает эти файлы и вызывает
`buildReferences()` / `buildResourceSection()`.

## Что дальше (черновик плана)

1. **Каркас страницы** — `references/index.html`, который
   подключает `references.css` + `references.js`, задаёт `lang`
   и рендерит секции `#references-grid`, `#special-grid`,
   `#boards-grid`.
2. **Категории** — навигация по направлениям из `refsData`
   (брендинг, упаковка, плакаты, иллюстрация, 3D и т.д.).
3. **Быстрый шеринг заказчику** — выбрать несколько карточек по
   разным проектам и сгенерировать одну ссылку-подборку.
4. **Фильтр по проектам** — показывать релевантные референсы
   под конкретную задачу заказчика.
5. **Дизайн-система** — переиспользовать токены из `ksu/css/tokens.css`,
   чтобы справочник выглядел в одном стиле с портфолио.
